# MyDrugs — Vision

MyDrugs is not a normal tech mod. It is a risk/reward survival mod about altered states, addiction, ritual crafting, and dangerous power.

The player should constantly ask:

> Is this effect worth the consequence?

## Core fantasy

The player discovers substances, learns how they affect the body and mind, then uses them as tools to access power, rituals, machines, and strange experiences that vanilla Minecraft does not offer.

The mod should feel primitive, bodily, unstable, sacred, and sometimes funny, but never like a plain industrial recipe chain.

## Design pillars

1. **Drugs are tools, not collectibles.** Each substance must have a clear gameplay reason to exist.
2. **Power has consequences.** Strong effects should create risk through addiction, dose instability, withdrawal, crashes, bad trips, or ritual danger.
3. **Rituals feel sacred and unstable.** The Psy Mixer should feel like a small religious altar, not a normal machine.
4. **The player must understand feedback.** Effects, withdrawal, dose symptoms, ritual stability, and machine bonuses need readable HUD, GUI, sound, particles, or guide feedback.
5. **The mod must not become vanilla with more recipes.** The best features should create new decisions, new perception, new risks, or new forms of exploration.

## First playable target

The first strong playable version should prove this loop:

```text
Find plants
→ process them with primitive tools
→ consume or ritualize them
→ gain useful but risky effects
→ build the Psy Mixer
→ create first ritual variants
→ manage addiction, withdrawal, and dose
→ use the result to progress further
```

## Tone

- Mystical, bodily, strange, and Minecraft-like.
- Religious language is allowed for Psy Mixer/gods messages.
- Humor is fine, but the core ritual tone should stay serious and slightly intimidating.
- Avoid real-world chemistry instructions. Use fictional, symbolic, or gameplay wording.

## What to avoid

- Endless machine chains with no player decision.
- Effects that are powerful but invisible.
- Punishments that feel random and unreadable.
- Features that only exist because they are realistic, not because they are fun.
- Large AI-generated rewrites that mix unrelated systems.
